# 审查示例

## 示例一：不合规岗位描述

### 输入

```
岗位名称：地区经理（OTC）
岗位职责：
1. 指导带领代表开发连锁药店、单体及诊所等终端，帮助代表制定区域销售计划，确保代表完成销售指标
2. 负责区域内产品的市场推广和销售工作
3. 维护重点客户关系，定期拜访核心终端
4. 完成公司下达的销售任务和回款目标
```

### 审查结果

**合规判定**：❌ 不合规
**风险等级**：高风险

| # | 违反规则 | 严重性 | 违规证据 | 说明 | 修改建议 |
|---|----------|--------|----------|------|----------|
| 1 | 身份偏移逻辑 | 高 | "确保代表完成销售指标" | 将医药代表定位为销售角色，设定销售指标 | 改为"确保代表完成学术覆盖目标" |
| 2 | 身份偏移逻辑 | 高 | "负责区域内产品的市场推广和销售工作" | 明确将销售作为岗位职责 | 改为"负责区域内产品的学术推广工作" |
| 3 | 身份偏移逻辑 | 高 | "完成公司下达的销售任务和回款目标" | 销售任务和回款属于商业行为 | 删除此条或改为学术推广相关指标 |

## 示例二：合规岗位描述

### 输入

```
岗位名称：医学信息专员
岗位职责：
1. 负责区域内目标医院的学术推广工作
2. 向医务人员传递产品相关的学术信息和最新研究进展
3. 组织和参与区域内的学术会议、科室会等学术活动
4. 收集并反馈药品临床使用情况及不良反应信息
5. 建立和维护与医学专家的学术交流关系
```

### 审查结果

**合规判定**：✅ 合规
**风险等级**：无风险

岗位名称使用"医学信息专员"，职责描述均围绕学术推广、信息传递、不良反应收集等合规范围内的活动，未涉及销售、收款、处方干预等违规行为。

## 示例三：制度文件（灰色地带）

### 输入

```
列名费用申请与使用原则：
1. 列名费用用于推动区域销量增长
2. 费用标准根据处方目标完成情况确定
3. 需提供终端拜访记录和学术活动证明
4. 费用核销需经合规审核
```

### 审查结果

**合规判定**：❌ 不合规
**风险等级**：中风险

| # | 违反规则 | 严重性 | 违规证据 | 说明 | 修改建议 |
|---|----------|--------|----------|------|----------|
| 1 | 利益交换逻辑 | 高 | "列名费用用于推动区域销量增长" | 将费用用途与销量增长直接关联，可能构成以费用换取商业产出的安排 | 改为"列名费用用于合规学术交流活动支持" |
| 2 | 处方干预逻辑 | 高 | "费用标准根据处方目标完成情况确定" | 将费用标准与处方目标挂钩，可能影响医生独立处方行为 | 删除处方目标相关表述，改用合规的学术活动质量或覆盖指标 |

**说明**：虽然文件提到了"学术活动证明"和"合规审核"，但费用用途和标准仍直接关联销量、处方目标，建议修改为与合规学术活动相关的预算和审核要求。
